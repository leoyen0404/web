# 運算思維期末考友善版筆記

這份是給「平常沒怎麼摸 Django、看到 `urls.py` 會先蛤一聲」的人看的。  
目標不是變工程師，是明天考試看到一大包程式碼時，知道它們在演哪齣，能把流程講出來。

## 0. 這門課期末到底在考什麼

老師很可能不是要你手寫完整網站，而是給你很多頁 code，問你：

- 使用者按了某個按鈕，為什麼畫面會變？
- 使用者輸入帳號密碼，為什麼 server 端某個 function 會被呼叫？
- 帳號密碼是從哪裡送出去的？
- server 查 database 後，結果怎麼回到 browser？
- `models.py` 改了，為什麼還要 `makemigrations` 和 `migrate`？
- 為什麼刪掉學生，他的成績也一起消失？

所以你要會的是「講故事」：

> 使用者做一個動作，browser 送 request 給 server，Django 根據網址決定呼叫哪個 function，function 可能查 database 或算東西，最後把 HTML 或 JSON 回給 browser，browser 再顯示結果。

先背這句，真的很有用。

## 1. 先認識三個角色

### User

人。也就是你、老師、使用者。

User 會做的事：

- 在網址列輸入網址
- 按 button
- 填帳號密碼
- 送出表單

### Web client

瀏覽器，例如 Chrome、Safari、Firefox。

它負責：

- 顯示網頁
- 幫 user 把 request 送給 server
- 收到 server 回來的 response 後顯示畫面

### Web server

後端程式。這堂課後半常用 Django。

它負責：

- 接 request
- 決定要呼叫哪段 Python code
- 查 database 或做計算
- 回傳 HTML 或 JSON

## 2. URL 是什麼

例子：

```text
http://127.0.0.1:8000/check-grade/
```

可以拆成：

```text
http://  127.0.0.1  :8000   /check-grade/
協定      IP          port    path
```

### IP

要找哪一台電腦。

`127.0.0.1` 或 `localhost` 代表「我自己這台電腦」。

### Port

同一台電腦可以開很多服務，所以 port 是「要找這台電腦上的哪個服務」。

Django 預設常用：

```text
8000
```

### Path

要 server 的哪個功能。

例如：

```text
/check-grade/
/mood/
/allMessages/
/addNumbers/
```

考試常問 path，請記得：path 就是網址 IP 和 port 後面的那段。

---

## 3. Django 是什麼感覺

你可以把 Django 想成一間餐廳。

- Browser：客人
- Request：點餐單
- `urls.py`：櫃台，看你點哪道菜，決定叫哪個廚師
- `views.py`：廚師，真正處理事情
- `models.py`：資料庫菜單/倉庫規格，定義有哪些資料表
- `templates/*.html`：最後端給客人的盤子/頁面
- `db.sqlite3`：真正存資料的倉庫

所以當你看到：

```python
path("check-grade/", LearnerCheckGrade)
```

不要怕。人話就是：

> 如果 browser 要 `/check-grade/` 這個 path，就請 Django 去呼叫 `LearnerCheckGrade` 這個 Python function。

看到：

```python
path("mood/", Mood)
```

人話就是：

> 如果 browser 要 `/mood/`，就呼叫 `Mood`。

看到：

```python
path("LearnerLogin/", LearnerLogin)
```

人話就是：

> 如果 browser 要 `/LearnerLogin/`，就呼叫 `LearnerLogin`。

這就是 `urls.py` 的核心。

## 4. 一個 Django request 的基本流程

請背這個流程：

1. User 在 browser 做一個動作
2. Browser 送 request 給 Django server
3. Django 看 `urls.py`
4. `urls.py` 根據 path 找到對應 function
5. Django 執行 `views.py` 裡的 function
6. function 可能查 database、算數字、檢查帳密
7. function 回傳 response
8. Browser 顯示結果

如果要講得像考試答案：

> Client 發出 request 後，Django 會根據 `urls.py` 裡的 `urlpatterns` 比對 request path。比對成功後，Django 呼叫對應的 view function。view function 處理 request，可能存取 database 或 render template，最後產生 response 回傳給 client。

## 5. `render()` 是什麼

你會看到：

```python
return render(request, "Mood.html", {"title": "Mood by Jemini"})
```

人話：

> 請 Django 拿 `Mood.html` 這個網頁模板，順便把 `title` 這個資料塞進去，最後做成真正的 HTML 回給 browser。

HTML 裡可能有：

```html
<title>{{ title }}</title>
```

這裡的：

```text
{{ title }}
```

會被換成：

```text
Mood by Jemini
```

所以 `render()` 不是單純傳檔案，是「把 template 執行/加工後再傳回去」。

---

## 6. `HttpResponse()` 是什麼

你可能看到：

```python
return HttpResponse("Your name is : " + userName)
```

人話：

> Server 直接回傳一段文字或 HTML 給 browser。

它比 `render()` 簡單，沒有另外拿 template 來加工。

## 7. `JsonResponse()` 是什麼

你可能看到：

```python
return JsonResponse({
    "submissionOK": True,
    "serverMsg": serverMsg,
    "timeStr": timeStr,
})
```

人話：

> Server 不回整個網頁，而是回一包資料給 JavaScript。

這常跟 `fetch()` 搭配。

例子：

- Browser 上的 JS 用 `fetch()` 問 server：「這個帳密對嗎？」
- Server 回 JSON：「對，這是成績」或「錯，密碼錯」
- Browser 不一定換頁，只更新某個區塊

## 8. Template 是什麼

Template 通常是 `.html`，但它不是普通 HTML。

它可以有 Django 特殊語法。

### 變數

```html
{{ result }}
```

意思：

> 把 server 傳來的 `result` 顯示在這裡。

### 迴圈

```html
{% for m in messages %}
  <p>{{ m.user_name }}: {{ m.content }}</p>
{% endfor %}
```

意思：

> 對每一筆留言，都產生一段 HTML。

### CSRF token

```html
{% csrf_token %}
```

意思：

> Django 的安全機制。POST 表單常常要放，不然可能被 Django 擋掉。

## 9. GET 和 POST

### GET

常用在「查資料、不改資料」。

例子：

```text
/addNumbers/?a=10&b=20
```

server 可以用：

```python
num1 = request.GET.get("a", 0)
num2 = request.GET.get("b", 0)
```

意思：

> 從 URL 後面的 `?a=10&b=20` 拿資料。

### POST

常用在「送資料、改資料、登入、新增留言」。

server 可以用：

```python
form = MessageForm(request.POST)
```

或：

```python
data = json.loads(request.body)
```

意思：

> 從 request body 裡拿資料。

簡單分：

- GET：資料常出現在網址上
- POST：資料藏在 request body，比較適合送表單、帳密、修改資料

## 10. JavaScript `fetch()` 是什麼

這是 browser 裡的 JavaScript 主動跟 server 要資料。

例子：

```javascript
fetch("/LearnerLogin/", {
  method: "POST",
  headers: {"Content-Type": "application/json"},
  body: JSON.stringify(dataToSend)
})
```

人話：

> Browser 用 POST 方法，把 `dataToSend` 這包資料送到 server 的 `/LearnerLogin/`。

`JSON.stringify(dataToSend)`：

> 把 JavaScript object 轉成 JSON 字串，這樣才能放進 request body 送出去。

Server 端：

```python
data = json.loads(request.body)
```

人話：

> 把 browser 送來的 JSON 字串轉回 Python dict。

## 11. 重要題型：Mood 網頁

看到 URL：

```text
/mood/
```

`urls.py`：

```python
path("mood/", Mood)
```

意思：

> request `/mood/` 時，呼叫 `Mood`。

View：

```python
def Mood(request):
    return render(request, "Mood.html", {"title": "Mood by Jemini"})
```

意思：

> 回傳 `Mood.html` 這個頁面。

HTML 裡有：

```html
<button onclick="changeMood()">切換狀態</button>
```

意思：

> User 按 button 時，browser 端會呼叫 JavaScript 的 `changeMood()`。

JavaScript：

```javascript
function changeMood() {
    const moods = [
        { msg: "Code 一次就過！", color: "#2ecc71" },
        { msg: "Bug 正在路上...", color: "#e74c3c" },
        ...
    ];

    const randomIndex = Math.floor(Math.random() * moods.length);
    const selected = moods[randomIndex];

    document.getElementById("message").innerText = selected.msg;
    document.body.style.backgroundColor = selected.color;
}
```

考試回答：

> 按下 button 會執行 `onclick="changeMood()"`，所以 browser 呼叫 `changeMood()`。這個 function 先建立一個 moods 陣列，裡面每個元素都有文字 `msg` 和顏色 `color`。接著用 `Math.random()` 產生亂數 index，選出其中一個 mood。最後用 `document.getElementById("message").innerText = selected.msg` 改變畫面文字，並用 `document.body.style.backgroundColor = selected.color` 改變背景色。

注意：

> 因為亂數可能抽到跟上次一樣的 mood，所以按 button 不一定每次看起來都有變。

## 12. 重要題型：Check Grade 登入查成績

這題最重要。請先看大地圖。

### 大地圖

1. User 打開 `/check-grade/`
2. Server 回傳一個有帳號密碼欄位的網頁
3. User 輸入 account/password
4. User 按 `Check Grades`
5. Browser 的 JavaScript 讀取 account/password
6. JavaScript 用 `fetch("/LearnerLogin/")` 把資料送到 server
7. Django `urls.py` 看到 `/LearnerLogin/`，呼叫 `LearnerLogin`
8. `LearnerLogin` 解析帳密
9. Server 查 database
10. Server 回 JSON
11. JavaScript 讀 JSON
12. Browser 顯示「成績」或「登入錯誤」

這個流程一定要會講。

### 12.1 為什麼 `/check-grade/` 會出現登入頁

`urls.py`：

```python
path("check-grade/", LearnerCheckGrade)
```

View：

```python
def LearnerCheckGrade(request):
    return render(request, "LearnerCheckGrade.html", {
        "title": "Learner Check Grade Page"
    })
```

人話：

> Browser request `/check-grade/`，Django 根據 `urls.py` 呼叫 `LearnerCheckGrade`。這個 function 用 `render()` 回傳 `LearnerCheckGrade.html`，所以 browser 顯示登入查成績頁。

### 12.2 為什麼按 Check Grades 會呼叫 server 的 `LearnerLogin`

HTML button：

```html
<button onclick="LearnerLogin()">Check Grades</button>
```

意思：

> 按 button 時，先呼叫 browser 裡 JavaScript 的 `LearnerLogin()`。

JavaScript 裡：

```javascript
fetch("/LearnerLogin/", {
  method: "POST",
  headers: {"Content-Type": "application/json"},
  body: JSON.stringify(dataToSend)
})
```

意思：

> JavaScript 送 POST request 到 `/LearnerLogin/`。

`urls.py`：

```python
path("LearnerLogin/", LearnerLogin)
```

意思：

> Django 收到 `/LearnerLogin/` 這個 request 後，會呼叫 server 端 Python function `LearnerLogin`。

高分答案：

> User 按下 `Check Grades` 後，button 的 `onclick` 先呼叫 client 端 JavaScript function `LearnerLogin()`。這個 JS function 裡有 `fetch("/LearnerLogin/", ...)`，所以 browser 送出 POST request 到 `/LearnerLogin/`。Django 的 `urls.py` 有 `path("LearnerLogin/", LearnerLogin)`，因此 server 端的 `LearnerLogin(request)` 會被呼叫。

### 12.3 帳號密碼在哪裡被讀取

HTML：

```html
<input type="text" id="acctInput">
<input type="password" id="passwordInput">
```

JavaScript：

```javascript
const dataToSend = {
  account: document.getElementById("acctInput").value,
  password: document.getElementById("passwordInput").value,
};
```

人話：

> `document.getElementById("acctInput").value` 讀取帳號欄位的值。  
> `document.getElementById("passwordInput").value` 讀取密碼欄位的值。

送出：

```javascript
body: JSON.stringify(dataToSend)
```

人話：

> 把 account/password 包成 JSON，放進 request body 送給 server。

### 12.4 Server 怎麼收到帳密

Python：

```python
data = json.loads(request.body)
account = data.get("account")
password = data.get("password")
```

人話：

> `request.body` 裡有 browser 送來的 JSON。`json.loads` 把它轉成 Python dict。然後用 `data.get("account")` 和 `data.get("password")` 取出帳號密碼。

### 12.5 登入錯誤怎麼發生

Server 會檢查：

```python
lgs = Learner.objects.filter(mID=acct)
```

人話：

> 去 Learner table 找有沒有這個帳號。

如果沒有：

```python
errorMsg = "There is no such learner account..."
```

如果有，但密碼錯：

```python
if learner.mPassword != passwd:
    errorMsg = "Wrong password..."
```

最後回：

```python
return {
    "submissionOK": False,
    "serverMsg": serverMsg,
    "timeStr": timeStr,
}
```

人話：

> Server 用 `submissionOK: False` 告訴 browser 登入失敗，錯誤原因放在 `serverMsg`。

Browser 端：

```javascript
if (submissionOK) {
  ...
} else {
  serverMsgContainer.innerHTML =
    `<h1>There was a log-in error :</h1>
     ${LearnerLogin.serverMsg}`;
}
```

人話：

> JS 看到 `submissionOK` 是 false，就顯示登入錯誤，並把 server 給的錯誤訊息印出來。

### 12.6 登入成功時，成績怎麼來

Python：

```python
grades = learner.grade_set.all()
```

人話：

> 找出這個學生所有成績。

為什麼可以這樣？

因為 `Grade` model 裡有：

```python
mLearner = models.ForeignKey(Learner, on_delete=models.CASCADE)
```

代表 Grade 跟 Learner 有關係，所以 Django 幫 Learner 自動產生 `grade_set` 這個反向查詢。

接著：

```python
for grade in grades:
    courses = Course.objects.filter(mID=grade.mCourse)
    courseName = courses[0].mName
    serverMsg += courseName + " : " + str(grade.mGrade) + " <BR>"
```

人話：

> 對每一筆成績，先找課程名稱，再把「課程名稱 : 分數」加到 `serverMsg` 裡。

最後：

```python
return {
    "submissionOK": True,
    "serverMsg": serverMsg,
    "timeStr": timeStr,
}
```

Browser 收到後：

```javascript
serverMsgContainer.innerHTML =
  `<h1>Grades of ${account} :</h1>
   ${LearnerLogin.serverMsg}`;
```

人話：

> Browser 把 server 傳回的成績字串放進網頁裡。

## 13. Database：不要被名詞嚇到

### Table

表格。

例如：

```text
Learner table
Course table
Grade table
```

### Row / record / tuple

表格裡的一筆資料。

例如一個學生、一門課、一筆成績。

### Column / field

表格裡的一個欄位。

例如：

```text
學號、姓名、密碼、分數
```

### Primary key

用來唯一辨認一筆資料的欄位。

例如：

```python
mID = models.CharField(primary_key=True)
```

人話：

> 這個欄位不能重複，因為它就是這筆資料的身分證。

### `unique=True`

```python
mID = models.CharField(max_length=30, unique=True)
```

人話：

> 這個欄位不能重複。

所以如果已經有學號 `11427301`，你再新增一次，就會錯。

## 14. ForeignKey 是什麼

先用人話：

> ForeignKey 就是「這筆資料跟另一張表的某筆資料有關」。

例如：

```python
class Course(models.Model):
    mInstructor = models.ForeignKey(Instructor, on_delete=models.CASCADE)
```

人話：

> 每門課都有一位老師，所以 Course 裡有一個欄位指向 Instructor。

再看：

```python
class Grade(models.Model):
    mLearner = models.ForeignKey(Learner, on_delete=models.CASCADE)
```

人話：

> 每筆成績都屬於某個學生，所以 Grade 裡有一個欄位指向 Learner。

---

## 15. `on_delete=models.CASCADE`

```python
mLearner = models.ForeignKey(Learner, on_delete=models.CASCADE)
```

人話：

> 如果某個學生被刪掉，跟他有關的成績也一起刪掉。

為什麼合理？

因為成績必須屬於某個學生。如果學生不存在了，留下他的成績會變成沒有主人、沒有意義的資料。

考試回答：

> `on_delete=models.CASCADE` 表示當被參照的資料被刪除時，參照它的資料也會被一起刪除。因為 `Grade.mLearner` 是指向 `Learner` 的 foreign key，所以刪除 learner 時，該 learner 對應的 grade rows 也會被 cascade delete，以維持資料一致性。

## 16. `course_set` / `grade_set` 是什麼

你可能看到：

```python
i.course_set.create(mID="111", mName="資料結構")
```

或：

```python
learner.grade_set.all()
```

人話：

> 因為 Course 有 ForeignKey 指向 Instructor，所以 Django 讓 instructor 可以反過來找到他的 courses，名字叫 `course_set`。

> 因為 Grade 有 ForeignKey 指向 Learner，所以 Django 讓 learner 可以反過來找到他的 grades，名字叫 `grade_set`。

所以：

```python
learner.grade_set.all()
```

意思是：

> 找出這位學生的所有成績。

## 17. `models.py` 和 `db.sqlite3`

`models.py`：

> 用 Python class 定義 database 應該長什麼樣。

`db.sqlite3`：

> 真正存資料的 database 檔案。

例子：

```python
class Message(models.Model):
    user_name = models.CharField(max_length=20)
    content = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)
```

人話：

> Django 會根據這個 class 建立一張 Message table，裡面有 user_name、content、created_at 這些欄位。

考試回答：

> `models.py` 是 Python 端對 database schema 的描述，真正的資料存在 `db.sqlite3`。每個繼承 `models.Model` 的 class 通常對應 database 裡的一張 table，class 裡的 fields 對應 table 的 columns，而每個 model instance 對應 table 的一筆 row。

---

## 18. 為什麼要 `makemigrations` 和 `migrate`

改了 `models.py` 後，database 不會自動改。

你要做：

```bash
./manage.py makemigrations
./manage.py migrate
```

### `makemigrations`

人話：

> 幫你根據 `models.py` 的改變，寫出一份「等等要怎麼改 database」的計畫書。

### `migrate`

人話：

> 真正執行那份計畫書，去修改 `db.sqlite3`。

考試回答：

> 修改 `models.py` 只是修改 Python 程式碼，database schema 尚未真正改變。`makemigrations` 會根據 model 的變化產生 migration file，而 `migrate` 會執行 migration，真正建立或修改 `db.sqlite3` 裡的 table schema。

## 19. 為什麼新增欄位有時會失敗

如果原本 Learner table 已經有很多學生。

你突然加：

```python
mPassword = models.CharField(max_length=30, null=False)
```

問題：

> 舊學生本來沒有 password，現在你要求 password 不能是空的。那舊資料要填什麼？

所以 migration 可能不接受。

改成：

```python
mPassword = models.CharField(max_length=30, default="")
```

就可以。

因為：

> 舊資料如果沒有 password，就先補空字串。

## 20. Form 是什麼

Django Form 是幫你處理表單的工具。

例如：

```python
class TwoIntForm(forms.Form):
    a = forms.IntegerField(label="數字 A")
    b = forms.IntegerField(label="數字 B")
```

人話：

> 這個表單需要兩個整數欄位，叫 a 和 b。

View：

```python
aForm = TwoIntForm(request.GET)

if aForm.is_valid():
    sum = aForm.cleaned_data["a"] + aForm.cleaned_data["b"]
```

`is_valid()` 人話：

> Django 幫你檢查 user 輸入是不是符合規定。

如果 `a`、`b` 真的都是整數，才會進入 `if`。

`cleaned_data` 人話：

> 檢查通過後，Django 幫你整理好的資料。原本 browser 傳來的是字串，這裡已經變成 Python int。

## 21. ModelForm 是什麼

```python
class MessageForm(forms.ModelForm):
    class Meta:
        model = Message
        fields = ["user_name", "content"]
```

人話：

> 根據 Message 這張 table/model，自動做一個可以填 user_name 和 content 的表單。

常見流程：

1. User 打開留言板
2. Server 給一個空表單
3. User 填留言並按送出
4. Browser 送 POST request
5. Server 用 `MessageForm(request.POST)` 接收資料
6. `form.is_valid()` 檢查
7. `form.save()` 存到 database
8. `redirect()` 讓 browser 重新讀頁面
9. 新留言出現在歷史留言中

`form.save()` 很重要：

> 它會根據 ModelForm 的資料新增或修改 database row。

## 22. 留言板 `allMessages/`

View：

```python
def DisplayMessages(request):
    all_msgs = Message.objects.all().order_by("-created_at")
    return render(request, "allMessages.html", {"messages": all_msgs})
```

人話：

1. 從 Message table 撈所有留言
2. 依時間新到舊排序
3. 把資料取名叫 `messages`
4. 丟給 `allMessages.html`

Template：

```html
{% for m in messages %}
  <strong>{{ m.user_name }}</strong>
  <p>{{ m.content }}</p>
  <small>{{ m.created_at }}</small>
{% endfor %}
```

人話：

> 對每一筆 message，顯示名字、內容、時間。

考試回答：

> Server 在 view function 中用 `Message.objects.all().order_by("-created_at")` 從 database 取出所有留言，並透過 `render(..., {"messages": all_msgs})` 傳給 template。Template 使用 `{% for m in messages %}` 逐筆 access 留言，再用 `{{ m.user_name }}`、`{{ m.content }}`、`{{ m.created_at }}` 顯示欄位內容。

## 23. 加數字 `addNumbers/`

View：

```python
num1 = request.GET.get("a", 0)
num2 = request.GET.get("b", 0)

try:
    total = int(num1) + int(num2)
except ValueError:
    total = "請輸入有效的數字"
```

人話：

> 從 URL query string 拿 a 和 b，轉成整數後相加。如果轉不了，顯示錯誤訊息。

Template：

```html
<input type="number" name="a" value="{{ num1 }}">
<input type="number" name="b" value="{{ num2 }}">
<h2>計算結果：{{ result }}</h2>
```

重點：

- input 的 `name="a"` 對應 `request.GET.get("a")`
- input 的 `name="b"` 對應 `request.GET.get("b")`
- browser 傳來通常是字串，所以要 `int()`
- `{{ result }}` 顯示後端算好的結果

## 25. 外部連 Django server

平常：

```bash
./manage.py runserver
```

通常只給本機測試。

若要讓外部也能連：

```bash
./manage.py runserver 0.0.0.0:54321
```

人話：

- `0.0.0.0`：接受從其他 IP 來的連線
- `54321`：指定 server 使用的 port

別人可能用：

```text
http://192.168.0.4:54321/iiiindex
```

拆解：

- `192.168.0.4`：server 的 IP
- `54321`：port
- `/iiiindex`：path

## 26. VirtualBox / Ubuntu / SSH 超簡版

VirtualBox：

> 在你的 Windows/Mac 裡面跑一台假的電腦。

Ubuntu：

> 那台假的電腦裡裝的 Linux 系統。

為什麼不用 WSL 就好？

> 因為 VirtualBox 裡的 Ubuntu 更像真正 server，尤其網路、SSH、防火牆比較接近實務環境。

SSH：

> 從 Windows/Mac 連進 Ubuntu 的遠端登入方式。

常見指令：

```bash
ssh username@192.168.x.x
ssh -p 2222 username@192.168.x.x
```

防火牆：

```bash
sudo ufw allow ssh
sudo ufw allow 2222/tcp
sudo ufw status
```

判斷連不上：

- `ping` 不到：網路可能不通
- `Connection refused`：機器到了，但服務沒開或 port 不對
- `Permission denied`：帳號密碼錯
- `Timeout`：可能網路或防火牆擋住

## 27. 最後背這些萬用句

### 萬用句 1：解釋 request 流程

> User 在 browser 做出某動作後，browser 會送 request 給 Django server。Django 根據 request 的 path 到 `urls.py` 裡比對 `urlpatterns`，找到對應的 view function。該 function 處理 request，可能讀取 request 裡的資料、查 database 或進行計算，最後回傳 HTML 或 JSON response 給 browser。

### 萬用句 2：解釋 `render`

> `render()` 會指定一個 template 檔案，並把 context dict 裡的資料交給 template engine。Template engine 會把 `{{ ... }}` 和 `{% ... %}` 這些 Django template 語法處理成一般 HTML，最後回傳給 client。

### 萬用句 3：解釋 `fetch`

> `fetch()` 是 client 端 JavaScript 主動送 request 給 server 的方式。它可以指定 URL、method、headers 和 body。若 body 使用 `JSON.stringify(...)`，server 端通常會用 `json.loads(request.body)` 解析資料。

### 萬用句 4：解釋 Model 和 database

> Django 的 model class 是 database schema 的 Python 描述。每個 model class 通常對應一張 table，field 對應 column，model instance 對應 row。真正資料存在 `db.sqlite3`。

### 萬用句 5：解釋 migration

> 修改 `models.py` 後，database 不會自動改變。`makemigrations` 會產生描述 schema 變更的 migration file，`migrate` 才會真正執行這些變更並修改 database。

### 萬用句 6：解釋 foreign key

> ForeignKey 表示一張 table 的資料參照另一張 table 的資料，用來建立資料之間的關係。Django 也會提供反向查詢，例如 `grade_set`，讓 learner 可以找到它相關的 grades。

### 萬用句 7：解釋 cascade delete

> `on_delete=models.CASCADE` 表示被參照的資料刪除時，參照它的資料也會一起刪除，這樣可以避免留下沒有主人的資料並維持 database consistency。
