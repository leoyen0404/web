## 1. 大段 code dump 解題法

考卷如果給很多頁 code，不要從第一行慢慢讀。用這個順序：

1. 找 URL path
   - 看 `mysite/urls.py`
   - 找 `path("xxx/", SomeFunction)`
   - 這決定「client request 某 path 時 server 會呼叫哪個 function」

2. 找 view function
   - 看 `myapp/views.py`
   - 判斷它回傳 `render`、`HttpResponse`、`JsonResponse`、`redirect`
   - `render(request, "xxx.html", context)` 代表 server 會執行 template engine，產生 HTML response

3. 找 template
   - 看 `myapp/templates/*.html`
   - 找 `{{ title }}`、`{{ result }}`、`{% for m in messages %}`、`onclick=...`、`form method=... action=...`
   - 這決定瀏覽器畫面怎麼產生，以及 user 按 button 後會發生什麼事

4. 找資料來源
   - `request.GET.get(...)`：資料來自 URL query string 或 GET form
   - `request.POST.get(...)`：資料來自 POST form body
   - `json.loads(request.body)`：資料來自 JavaScript `fetch` 送出的 JSON body
   - `Model.objects...`：資料來自 database table

5. 找資料回到 client 的方式
   - `render(..., {'name': value})`：後端變數進 template，變成 HTML
   - `JsonResponse({'key': value})`：後端回 JSON，前端 JS 用 `.then(response => response.json())` 取值
   - `HttpResponse("text")`：直接回一段文字/HTML

6. 回答題目時要寫「因果鏈」
   - 不要只說「因為有 fetch」
   - 要說：button `onclick` 呼叫 JS function；JS function 讀 input；`fetch("/LearnerLogin/", {method:"POST", body: JSON.stringify(...)})` 送 request；`urls.py` 把 path 對到 `LearnerLogin`; view 用 `json.loads(request.body)` 拿資料；server 回 `JsonResponse`; client `.then` 解析 JSON；最後用 `innerHTML` 改畫面。

## 2. Django 專案骨架

常見建立流程：

```bash
pip3.11 install django
django-admin startproject mysite
cd mysite
./manage.py startapp myapp
./manage.py runserver
```

重要檔案：

- `manage.py`：Django 管理入口，跑 server、migration、shell 等
- `mysite/settings.py`：整個 project 設定，包含 `INSTALLED_APPS`
- `mysite/urls.py`：URL routing 總表
- `myapp/views.py`：處理 request 的 function
- `myapp/models.py`：用 Python class 定義 database schema
- `myapp/forms.py`：表單驗證與 HTML form 產生規則
- `myapp/templates/*.html`：Django template
- `myapp/admin.py`：註冊 model 到 admin 後台
- `db.sqlite3`：實際 SQLite database 檔案

答題模板：

> Django 將 project 與 app 分開。`mysite` 管整體設定與 routing，`myapp` 放某一組功能。當 client 發 request，Django 先看 `urls.py` 的 `urlpatterns`，決定呼叫哪個 view。view 再決定要查資料庫、計算、render template 或回 JSON。

## 3. URL routing：server 如何知道要呼叫哪個 function

典型程式：

```python
from django.urls import path
from myapp.views import *

urlpatterns = [
    path("admin/", admin.site.urls),
    path("", home_page),
    path("allMessages/", DisplayMessages),
    path("addNumbers/", AddNumbers),
    path("inputFormCalc/", InputFormCalc),
    path("modelFormCalc/", ModelFormCalc),
    path("mood/", Mood),
    path("check-grade/", LearnerCheckGrade),
    path("LearnerLogin/", LearnerLogin),
]
```

高分回答：

- `urlpatterns` 是 URL path 與 view function 的對照表
- client request `http://127.0.0.1:8000/allMessages/`
- path 部分是 `allMessages/`
- Django 找到 `path("allMessages/", DisplayMessages)`
- 所以 server 呼叫 `DisplayMessages(request)`

注意：

- `http://127.0.0.1:8000/` 的 path 是空字串 `""` 或 `/`
- `http://127.0.0.1:8000/addNumbers/?a=10&b=20` 的 path 是 `addNumbers/`，`?a=10&b=20` 是 query string
- `path("admin/", admin.site.urls)` 交給 Django admin 系統處理

## 4. View function：server 如何產生 response

### 4.1 `HttpResponse`

```python
def HomePage(request):
    return HttpResponse("<h1>Hello</h1>")
```

意思：

- view 直接製造 response body
- browser 收到後顯示這段 HTML/text
- 適合簡單測試，不適合複雜頁面

### 4.2 `render`

```python
def home_page(request):
    return render(request, "index.html", {"title": "Hello Django"})
```

意思：

- Django 找到 template `index.html`
- 把 context dict 裡的資料交給 template
- template engine 將 `{{ title }}` 等變數替換成實際值
- 最後產生完整 HTML response 給 client

考試用語：

> `render()` 並不是單純讀檔回傳。它會讓 Django template engine「執行」該 template：處理 `{{ ... }}`、`{% ... %}` 這些 template 語法，產生一般 HTML，再回傳給 browser。

### 4.3 `JsonResponse`

```python
return JsonResponse({
    "submissionOK": True,
    "serverMsg": serverMsg,
    "timeStr": timeStr,
})
```

意思：

- server 回傳 JSON，不是整頁 HTML
- 通常配合 JavaScript `fetch`
- client 不換頁，而是用 JS 更新畫面某一部分

### 4.4 `redirect`

```python
return redirect("/modelFormCalc")
```

意思：

- server 要 browser 再發一次新的 GET request
- 常用在 POST 成功後，避免重新整理時重複送出表單
- ModelForm 留言板中：POST 新留言成功 → `form.save()` → redirect → GET 頁面 → 撈出包含新留言的歷史留言

## 5. Template：`.html` 為什麼像程式

Django template 是「會被 server 執行的 HTML-like 程式」。

常見語法：

```html
<title>{{ title }}</title>
```

`{{ title }}` 會被 context 裡的 `title` 值替換。

```html
{% for m in messages %}
  <p>{{ m.user_name }}: {{ m.content }} ({{ m.created_at }})</p>
{% empty %}
  <p>目前還沒有留言喔！</p>
{% endfor %}
```

意思：

- `messages` 是 view 傳進來的資料
- template 逐筆取出 message
- 每一筆都產生一段 HTML
- `{{ m.user_name }}`、`{{ m.content }}`、`{{ m.created_at }}` 分別 access model instance 的欄位

表單常見：

```html
<form method="POST">
  {% csrf_token %}
  {{ form.as_p }}
  <button type="submit">發布留言</button>
</form>
```

重點：

- `method="POST"`：送出會變成 POST request
- `{% csrf_token %}`：Django 的 CSRF 防護，POST 表單常需要
- `{{ form.as_p }}`：讓 Django Forms 自動產生 input HTML

## 6. GET、POST、form、fetch

### 6.1 GET

例子：

```html
<form method="GET" action="">
  <input type="number" name="a">
  <input type="number" name="b">
  <button type="submit">計算</button>
</form>
```

按下後 URL 可能變成：

```text
http://127.0.0.1:8000/addNumbers/?a=10&b=20
```

server 取值：

```python
num1 = request.GET.get("a", 0)
num2 = request.GET.get("b", 0)
```

GET 特性：

- 資料放在 URL query string
- 適合查詢、搜尋、計算這類不改變 database 的 request
- 不適合密碼或大量資料

### 6.2 POST form

```html
<form method="POST">
  {% csrf_token %}
  {{ form.as_p }}
  <button type="submit">發布留言</button>
</form>
```

server 取值：

```python
form = MessageForm(request.POST)
```

POST 特性：

- 資料放在 request body
- 適合新增、修改、刪除 database
- Django 預設會檢查 CSRF token

### 6.3 JavaScript `fetch` + JSON

前端：

```javascript
const dataToSend = {
  account: document.getElementById("acctInput").value,
  password: document.getElementById("passwordInput").value,
};

fetch("/LearnerLogin/", {
  method: "POST",
  headers: {"Content-Type": "application/json"},
  body: JSON.stringify(dataToSend)
})
.then(response => response.json())
.then(obj => {
  // obj.submissionOK, obj.serverMsg, obj.timeStr
});
```

後端：

```python
data = json.loads(request.body)
account = data.get("account")
password = data.get("password")
return JsonResponse(responseAttr)
```

重點：

- `fetch("/LearnerLogin/")` 決定 client 要 request 哪個 path
- `body: JSON.stringify(dataToSend)` 將 JS object 轉成 JSON 字串送給 server
- `json.loads(request.body)` 在 Python server 端把 JSON 字串轉回 dict
- `JsonResponse` 把 Python dict 轉成 JSON response
- `response.json()` 在 browser 端把 JSON response 轉成 JS object

## 7. Django + DB：Model 與 SQLite

### 7.1 `models.py` 與 `db.sqlite3` 的關係

典型 model：

```python
class Message(models.Model):
    user_name = models.CharField(max_length=20)
    content = models.TextField()
    created_at = models.DateTimeField(auto_now_add=True)

    def __str__(self):
        return f"{self.user_name}: {self.content[:10]}..."
```

觀念：

- `models.py` 是 Python 端對 database schema 的描述
- `class Message(models.Model)` 對應到 SQLite 裡的一張 table，例如 `myapp_message`
- `user_name`、`content`、`created_at` 對應 table columns
- 每一個 `Message` instance 對應 table 的一列 row / tuple / record
- `db.sqlite3` 是實際存資料的 database file

高分答法：

> `models.py` 本身不是 database。它定義 Django ORM 應該建立與操作什麼樣的 database table。真正的 table 和資料在 `db.sqlite3` 裡。Django 透過 migration 把 Python class/field 的定義轉成 database schema。

### 7.2 為什麼改 `models.py` 後要 `makemigrations` / `migrate`

```bash
./manage.py makemigrations
./manage.py migrate
```

差別：

- `makemigrations`：根據 `models.py` 的改變產生 migration `.py` 檔，描述 schema 要怎麼改
- `migrate`：執行 migration，真的修改 `db.sqlite3` 的 table schema

考試句：

> 改 `models.py` 只是改 Python 程式碼，database file 還沒改。必須先 `makemigrations` 產生「改 DB schema 的 Python 程式」，再 `migrate` 執行它，`db.sqlite3` 裡的 table 才真的被建立或修改。

### 7.3 DB Browser 為什麼看不到最新變化

如果 DB Browser for SQLite 已經開著 `db.sqlite3`，後端又改了資料：

- DB Browser 可能顯示的是已載入/快取的舊狀態
- Close Database / Open Database 後重新讀取檔案
- 才會看到 Django 寫入的最新內容

## 8. Django ORM：查詢、新增、排序

常見操作：

```python
Message.objects.all()
Message.objects.all().order_by("-created_at")
Learner.objects.filter(mID=acct)
Course.objects.filter(mID=grade.mCourse)
Learner.objects.create(mID="11427301", mName="張三丰")
```

解釋：

- `objects` 是 Django model manager
- `.all()` 取出所有 rows
- `.filter(...)` 取出符合條件的 rows，結果是 QuerySet
- `.create(...)` 新增一筆 row
- `.order_by("-created_at")` 依時間由新到舊排序

注意：

- `filter()` 即使只有一筆，也回 QuerySet
- 取第一筆常用 `[0]`
- 若 QuerySet 長度為 0，取 `[0]` 會 error

## 9. Primary key、unique、default、null

### 9.1 Django 自動 primary key

若 model 沒指定 primary key：

```python
class Learner(models.Model):
    mID = models.CharField(max_length=30, unique=True)
```

Django 會自動建立 `id` 欄位作 primary key。

### 9.2 自己指定 primary key

```python
class Instructor(models.Model):
    mID = models.CharField(max_length=30, primary_key=True)
```

此時：

- `mID` 是 primary key
- 不會再自動加 `id`
- primary key 必須唯一且不可為 null

### 9.3 `unique=True`

```python
mID = models.CharField(max_length=30, null=False, unique=True)
```

意思：

- 此欄位不可重複
- 若新增相同 `mID`，SQLite 會丟 `UNIQUE constraint failed`

06/09 題型：

- `Learner.mID` 有 `unique=True`
- 所以新增同一個 `mID` 第二次會失敗
- `Learner1.mID` 沒有 unique，因此可重複

### 9.4 `null` 與 `default`

```python
mComment = models.CharField(max_length=30, null=True)
mComment = models.CharField(max_length=30, default="")
```

觀念：

- `null=True`：database 可存 NULL，因此新增 row 時可不給值
- `default=""`：沒給值時 Django/DB 可用預設值空字串
- `null=False` 且無 default：新增舊欄位到已有資料的 table 時會出問題，因為舊 rows 的新欄位不知道要填什麼

06/09 題型：

- 新增 `mPassword = models.CharField(max_length=30, null=False)` 時，既有 Learner rows 沒有 password 值，migration 不知道如何處理，所以不接受
- 改成 `default=""` 後，舊 rows 可用空字串補上，所以 migration 可成功

## 10. ForeignKey 與 relational database

### 10.1 範例 models

```python
class Instructor(models.Model):
    mID = models.CharField(max_length=30, primary_key=True)
    mName = models.CharField(max_length=30, null=False)

class Course(models.Model):
    mID = models.CharField(max_length=30, null=False, unique=True)
    mInstructor = models.ForeignKey(
        Instructor,
        on_delete=models.CASCADE,
        to_field="mID"
    )
    mName = models.CharField(max_length=30, null=False)

class Learner(models.Model):
    mID = models.CharField(max_length=30, null=False, unique=True)
    mName = models.CharField(max_length=30, null=False)

class Grade(models.Model):
    mLearner = models.ForeignKey(Learner, on_delete=models.CASCADE)
    mCourse = models.CharField(max_length=30, null=False)
    mGrade = models.IntegerField(default=-1)

    class Meta:
        constraints = [
            models.UniqueConstraint(
                fields=["mLearner", "mCourse"],
                name="uniqueness_of_learner_course"
            )
        ]
```

### 10.2 ForeignKey 的意思

`Course.mInstructor = ForeignKey(Instructor, ...)`：

- `Course` table 的某欄位會指向 `Instructor` table 的某筆資料
- 一位 instructor 可以有多門 course
- 一門 course 屬於一位 instructor

如果外鍵指向對方自動產生的 `id`，通常不用 `to_field`。若要指向 `Instructor.mID`，才需要 `to_field="mID"`。

### 10.3 Reverse relation：`course_set`、`grade_set`

如果有：

```python
class Course(models.Model):
    mInstructor = models.ForeignKey(Instructor, ...)
```

那麼某個 instructor instance `i` 可以：

```python
i.course_set.create(mID="111", mName="資料結構")
```

背後理性：

- 因為 `Course` 有 ForeignKey 指向 `Instructor`
- Django 自動提供反向關係 `course_set`
- `i.course_set.create(...)` 表示「新增一門 course，且它的 instructor 就是 i」
- 所以雖然指令從 instructor instance 出發，實際新增的是 `Course` table 的 row

同理：

```python
learner.grade_set.create(mCourse="098", mGrade=30)
```

表示替這位 learner 新增一筆 grade。

### 10.4 `on_delete=models.CASCADE`

意思：

- 若 referenced object 被刪掉，依附它的 rows 也一起刪

例子：

- `Grade.mLearner = ForeignKey(Learner, on_delete=models.CASCADE)`
- 刪掉某個 Learner
- 這個 learner 對應的 Grade rows 也被刪除

背後理性：

> 成績必須屬於某個學生。若學生 row 已不存在，留下他的成績 row 會變成孤兒資料，破壞資料一致性。因此 Django 依照 cascade 規則同步刪除相關資料。

### 10.5 `UniqueConstraint`

```python
models.UniqueConstraint(
    fields=["mLearner", "mCourse"],
    name="uniqueness_of_learner_course"
)
```

意思：

- 同一個 learner 在同一門 course 只能有一筆 grade
- 但同一 learner 可有不同 course
- 同一 course 可有不同 learner
- 這是「複合唯一限制」

## 11. Relational database / SQL 核心觀念

這部分適合用來解釋 Django model 背後的 database。

### 11.1 Table / Row / Column

可用 C struct 類比：

- `struct Student` 的每個 member 像 column / field
- `Student students[1000]` 像 table
- `students[0]` 像一筆 row / record / tuple

### 11.2 Primary key

Primary key：

- 唯一識別一筆 row
- 不可重複
- 不可為 null
- 例如學生學號、課程代號、Django 自動 `id`

### 11.3 Foreign key

Foreign key：

- 某 table 的欄位指向另一 table 的 primary key 或 unique 欄位
- 像 pointer/reference
- 用來表示 table 之間的關係

例：

- `Course.mInstructor` 指向 `Instructor.mID`
- `Grade.mLearner` 指向 `Learner.id`

### 11.4 為什麼要拆 table

若把學生、系所、課程、成績全部塞一張表：

- 資料重複
- 更新麻煩
- 容易不一致

拆成多張 table：

- `Student`
- `Department`
- `Course`
- `Grade`

再用 foreign key 連起來，能降低重複並維持一致性。

### 11.5 SQL CRUD

```sql
INSERT INTO Student (id, name, gpa) VALUES (1, 'Ann', 3.9);
SELECT name FROM Student WHERE gpa > 3.8;
UPDATE Student SET gpa = 4.0 WHERE id = 1;
DELETE FROM Student WHERE id = 1;
```

CRUD：

- Create：`INSERT`
- Read：`SELECT`
- Update：`UPDATE`
- Delete：`DELETE`

### 11.6 JOIN 與 aggregate

範例：

```sql
SELECT p.name, SUM(s.change_amount) AS current_stock
FROM Products p
JOIN Stock_Log s ON p.product_id = s.product_id
WHERE p.product_id = 101
GROUP BY p.name;
```

意思：

- `JOIN` 依 product id 把產品表與庫存紀錄表連起來
- `SUM(change_amount)` 把進貨/出貨紀錄加總
- 正數代表進貨，負數代表出貨
- `GROUP BY` 讓 aggregate 與產品名稱合理搭配

## 12. Django Forms 與 ModelForm

### 12.1 一般 Form

```python
from django import forms

class TwoIntForm(forms.Form):
    a = forms.IntegerField(label="數字 A")
    b = forms.IntegerField(label="數字 B")
```

View：

```python
def InputFormCalc(request):
    sum = None
    aForm = TwoIntForm(request.GET)

    if aForm.is_valid():
        sum = aForm.cleaned_data["a"] + aForm.cleaned_data["b"]

    return render(request, "addFormInt.html", {"form": aForm, "result": sum})
```

`is_valid()` 代表：

- Django 已依照 Form class 的欄位規則檢查資料
- `a`、`b` 必須存在或符合 required 規則
- 它們必須能轉成 integer
- 若 valid，轉型後的 Python int 存在 `cleaned_data`

容易考：

- `request.GET` 來的值本來是字串
- `IntegerField` 幫你做字串轉整數與錯誤檢查
- `cleaned_data["a"]` 才是可信、轉型後的資料

### 12.2 空 GET 為什麼可能顯示 form error

若 user 只輸入：

```text
http://127.0.0.1:8000/inputFormCalc/
```

## 20. Client / Server / User / IP / Port / Path

三個角色：

- User：人，按鍵盤、點 button、輸入 URL
- Web client：browser，例如 Chrome/Safari/Firefox
- Web server：Django/Flask server，接 request 回 response

User 做什麼會導致 request：

- 在網址列輸入 URL 按 Enter
- 點超連結

Request 常見部分：

- IP：要找哪台 machine，例如 `127.0.0.1`、`192.168.0.4`

- path：要該 server 的哪個資源/功能，例如 `/allMessages/`、`/LearnerLogin/`

`localhost` / `127.0.0.1`：

- 指 client 自己這台機器
- 在 Ubuntu Firefox 開 `localhost:8000` 是連 Ubuntu 自己的 Django server
- 從 Windows host 要連 Ubuntu VM，通常不能用 Windows 的 `localhost`，要用 Ubuntu VM IP

## 21. VirtualBox / Ubuntu / SSH / Firewall

用途：

- 建立接近真實 Linux server 的環境
- 比 WSL 更接近 production Linux 的網路與系統行為

VirtualBox network：

- NAT：VM 可上網，但外界不易直接連入
- Bridged Adapter：VM 像區網中獨立機器，有自己的 LAN IP，適合從 Windows host SSH 進去

SSH server：

```bash
sudo systemctl status ssh
```

Firewall：

```bash
sudo ufw allow ssh
sudo ufw allow 22/tcp

```

Windows host 連 Ubuntu：

```bash
ssh username@192.168.x.x
```

判斷問題：

- Ubuntu `ip a`：是否有 IP
- Ubuntu `ping 8.8.8.8`：是否能出網

- `Permission denied`：連到了，但帳密/權限錯

## 24. 常見問答模板

### Q1. Server 如何知道要回哪個 HTML？

答：

> Server 先由 `urls.py` 的 `path(...)` 決定呼叫哪個 view function。該 view function 內的 `render(request, "xxx.html", context)` 指定要使用哪個 template。Django template engine 執行 `xxx.html`，用 context 替換 template variables，產生 HTML response 回給 client。

### Q2. `.html` 如何 access server 傳來的資料？

答：

> View 把資料放進 context dict，例如 `{"messages": all_msgs}`。Template 中可用 `messages` 這個名字 access。若是 list/QuerySet，可用 `{% for m in messages %}` 逐筆處理；若是單一值，可用 `{{ result }}` 直接插入 HTML。

### Q3. 為什麼 `form.is_valid()` 重要？

答：

> 它讓 Django 根據 Form class 的欄位定義檢查 user input，包括必填、型別、格式。通過後資料會出現在 `cleaned_data`，且已轉成適當 Python 型別。例如 `IntegerField` 通過後拿到的是 int，不是原始字串。

### Q4. 為什麼要用 `redirect`？

答：

> POST 成功後 redirect 可讓 browser 再發 GET request 取得最新頁面，並避免 user 重新整理時重複送出 POST。留言板新增成功後 redirect，再重新撈 DB，就能顯示剛新增的留言。

### Q5. Server 如何知道 login 錯？

答：

> `LearnerLogin` 解析 request body 得到 account/password 後，呼叫 `LearnerLoginSupport`。`LearnerLoginSupport` 呼叫 `DoorCheck` 查 Learner table 並比對 password。若帳號不存在或密碼錯，回傳 `submissionOK: False` 與錯誤訊息 `serverMsg`。Client JS 讀到 `submissionOK` false 後，把錯誤訊息寫入頁面。

### Q6. 成績頁的 `serverMsg` 從哪來？

答：

> 登入成功後，server 端用 `learner.grade_set.all()` 取出該 learner 的所有成績。每筆 grade 再用 `Course.objects.filter(mID=grade.mCourse)` 找課程名稱，將「課名 : 分數」累加成字串 `serverMsg`。最後 `serverMsg` 被放入 JSON response，client 端用 `LearnerLogin.serverMsg` 顯示。
